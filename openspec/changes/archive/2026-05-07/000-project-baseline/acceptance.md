# Acceptance Criteria: 000-project-baseline

El baseline se considera correctamente establecido y aceptado si se cumplen los siguientes criterios:

- El archivo `agents.md` existe en la raíz y define las reglas.
- El archivo `changes.md` existe en la raíz e incluye el baseline.
- El archivo `specs/README.md` existe y explica la metodología.
- El archivo `specs/project.md` existe y documenta el estado actual.
- La carpeta `specs/changes/000-project-baseline` existe.
- Todos los artefactos del baseline (`proposal.md`, `design.md`, `tasks.md`, `acceptance.md`, `verification.md`) existen y contienen la información requerida.
- El estado actual del proyecto (Frontend, Backend, Arquitectura, API) queda claramente documentado.
- **NO SE MODIFICÓ ningún código fuente del proyecto** (frontend, backend, configs, dependencias).
